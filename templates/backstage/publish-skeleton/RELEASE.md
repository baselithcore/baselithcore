# Release runbook — {{ values.pluginName }}

Generated by the Backstage **Publish BaselithCore Plugin** template.

## Automated pipeline

1. Push a semver tag `vMAJOR.MINOR.PATCH`:

   ```bash
   git tag v{{ values.version }}
   git push origin {{ values.defaultBranch }} --tags
   ```

2. `.github/workflows/marketplace-publish.yml` picks up the tag and runs:
   - architecture boundary check
   - Bandit security scan
   - `baselith plugin marketplace validate .`
   - `baselith plugin marketplace publish .` against `{{ values.marketplaceUrl }}`
   - `gh release create vX.Y.Z` with auto-generated notes.

## Required secrets

| Secret                          | Where                 | Purpose                                    |
| ------------------------------- | --------------------- | ------------------------------------------ |
| `BASELITH_MARKETPLACE_TOKEN`    | repo/org secret       | Authenticates against the marketplace hub. |
| `GITHUB_TOKEN` (auto-provided)  | runtime               | Creates the GitHub release.                |

## Manual escape hatches

If the hub is unavailable or the repo still awaits approval, publish locally:

```bash
baselith plugin marketplace login --url {{ values.marketplaceUrl }}
baselith plugin marketplace publish .
```

## Release cadence

- **Semver**: enforce `MAJOR.MINOR.PATCH` via [`.releaserc.json`](./.releaserc.json).
- **Changelog**: auto-generated by `semantic-release` on `{{ values.defaultBranch }}`.
- **Catalog**: Backstage re-scrapes `catalog-info.yaml` every 5 minutes; tag
  pushes surface in the Software Catalog without manual intervention.
